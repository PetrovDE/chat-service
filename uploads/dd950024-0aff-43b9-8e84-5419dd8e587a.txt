1. Цель и задачи аналитического сервера

Будут скрипты на Python с оркестрацией через Airflow для ETL/ELT пайплайнов.

Хорошо если будет хранение и анализ логов.

Должен быть сбор и трансформация данных из разных источников.

Запуск Python- и SQL-скриптов аналитиками для анализа данных, формирования сегментов клиентов для CRM, настройка черных списков и ведение внутренней базы данныз

2. Компоненты, которые тебе нужны

✅ JupyterHub – для запуска и управления ноутбуками несколькими пользователями

✅ Apache Airflow – для оркестрации ETL/ELT пайплайнов

✅ Superset / Metabase / Redash – BI-инструменты

✅ ClickHouse и PostgreSQL в дадльнейшем при необходимости список расширим.

✅ Prometheus + Grafana – метрики, мониторинг

✅ MLflow / DVC / Kedro – если планируется MLOps

✅ Git + CI/CD – автоматизация, контроль версий. У нас есть корпроративный репозиторий, необходимо к нему настроить подключение.

✅ Spark / dbt / pandas / polars / SQL – нужные инструменты трансформации данных

Сколько пользователей будет - до 20 пользователей

Должна быть автоизация на Jupyterhub через учетную запись Active Directory

Нужно ли ограничение прав - на сам сервер аналитики заходить не будут, они пользвуются только web интерфейс сервисов

так же используются корпоративные сетевые диски
